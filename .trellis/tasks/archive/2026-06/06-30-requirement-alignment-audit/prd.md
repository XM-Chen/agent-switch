# 需求回溯与对齐审计

## 目标

回溯用户对 agent-switch 的原始需求、后续确认的范围变化与各子任务验收标准，并以当前仓库代码、任务文档、提交历史和可运行质量门为证据，判断项目是否：

1. 对齐用户需求；
2. 未完成或偏离哪些需求；
3. 在哪些方面超额完成；
4. 哪些超额实现属于合理演进，哪些可能增加了不必要复杂度或后续维护风险。

最终输出中文审计结论，并把审计依据沉淀在当前 Trellis 任务中。

## 已确认背景

- 用户明确询问：“你是否还记得我的需求，检查我的项目是否对齐或者超额完成了我的需求？”
- 当前任务只做需求回溯与项目对齐审计，不修改业务代码。
- 项目主线愿景来自父任务 `.trellis/tasks/archive/2026-06/06-26-agent-switch-web-router-mvp/prd.md`：agent-switch 是本地 agent 管理与路由应用，固定服务地址 `http://127.0.0.1:42567`，单进程单端口路径隔离，Tauri + Rust + Web 前端，中文 UI/文档，参考 `9router`、`ccs`、`cpa`、`sub2api`。
- 父任务把第一阶段拆成 8 个独立子任务：应用骨架、本地账号端点与凭据安全、模型管理刷新与别名、工具接管、路由故障转移、OpenAI-compatible v1 多端点、真实链路测试调试器、导入导出与设置。
- 后续又补充了 Dashboard 总览页任务，因为父任务 8 页信息架构中的“总览”页仍是占位。
- `trellis mem search` 找到历史会话摘要，确认用户曾要求“single local service on `http://127.0.0.1:42567`, single process + single port + path isolation”，并要求综合 `9router`、`ccs`、`cpa`、`sub2api`；还确认工具接管子任务范围仅为配置接管，不含当时尚未完成的代理路由。
- `trellis mem` 当前无法读取 OpenCode 平台历史日志；该限制会在审计结论中说明，避免把不可索引历史当作已覆盖证据。

## 审计需求

### R1 需求源回溯

- R1.1 汇总父任务与各子任务 PRD 中的用户需求、约束、验收标准和明确排除项。
- R1.2 区分“原始 MVP 必须完成”“后续用户确认扩展”“明确暂不纳入范围”“实现过程中新增的合理补强”。
- R1.3 引用证据时使用可追溯文件路径和行号，必要时补充 `trellis mem` 命中的历史会话片段。

### R2 当前项目现状检查

- R2.1 检查代码结构、前端页面、后端路由、数据库迁移、服务层、API 客户端与关键功能实现，判断每项需求是否已有对应实现。
- R2.2 检查近期提交与任务归档状态，确认哪些工作已完成、哪些只是文档承诺或占位。
- R2.3 运行非破坏性质量门（如 `npm run build`、`cargo check`、必要时 `cargo fmt --check` / `cargo clippy`），以验证“已实现”不是仅静态存在。
- R2.4 审计不得调用真实上游 API、不得写入用户真实 Claude Code / Codex / OpenCode 配置、不得修改业务代码。

### R3 对齐与超额完成判断

- R3.1 对每个主要需求域给出状态：已对齐、部分对齐、未对齐、超额完成、无法验证。
- R3.2 对超额完成项判断其性质：合理超额、可能过度工程、需要产品确认的扩展。
- R3.3 对未对齐/无法验证项给出影响、证据、建议下一步。
- R3.4 审计结论使用中文，并优先给出决策级摘要，再给出证据矩阵。

## 验收标准

- [ ] AC1：产出一份中文审计报告，覆盖父任务、8 个子任务和 Dashboard 补充任务的需求域。
- [ ] AC2：报告明确列出“对齐需求”“未完成/偏离需求”“超额完成需求”“无法验证需求”。
- [ ] AC3：每个重要判断都有证据来源：任务文档路径/代码路径/提交或命令输出。
- [ ] AC4：报告明确说明审计边界：不调用真实上游、不写真实工具配置、不修改业务代码。
- [ ] AC5：运行并记录非破坏性验证命令结果；若命令失败，报告中如实说明失败输出和影响。
- [ ] AC6：审计产物可用于用户决定下一步：继续补缺、收敛范围、或进入发布前打磨。

## 暂不纳入范围

- 不修复审计发现的问题。
- 不新增业务功能或重构代码。
- 不执行真实上游链路测试，不消耗用户 API quota。
- 不改写本机 Claude Code / Codex / OpenCode 配置。
- 不把 OpenCode 平台历史日志当作已覆盖证据，因为当前 `trellis mem` 明确提示 OpenCode reader 暂不可用。

## 已确认决策

- 审计深度：**允许运行本地非破坏性验证命令**（用户已确认）。可运行 `npm run build`、`cargo check`、`cargo fmt --check`、`cargo clippy`，以及只读检查命令（`git log`、目录/文件读取、`trellis mem` 检索）。禁止调用真实上游、禁止写真实工具配置、禁止修改业务代码。
