# Agent Platform Guidelines

> Project-specific guidance for agent-facing prompts, hooks, commands, and generated platform configuration.

---

## Overview

This repository does not contain a browser frontend, React components, or CSS assets. In this spec layer, "frontend" means the AI-facing platform surface: Markdown agent definitions, slash commands, skill docs, JSON/TOML settings, Python/JS/TS hooks, and plugin adapters that present Trellis workflow state to humans and agents.

Reference directories:
- `AGENTS.md` — cross-platform project instruction block.
- `.claude/` — Claude Code agents, skills, commands, settings, and hooks.
- `.cursor/`, `.codex/`, `.opencode/`, `.pi/`, `.reasonix/`, `.trae/`, `.zcode/` — platform-specific generated adapters.
- `.trellis/agents/` — channel-runtime agent definitions.
- `.trellis/workflow.md` — shared workflow text consumed by platform hooks.

## Pre-Development Checklist

Before modifying agent-platform files:

1. Identify the target platform directory and whether the file is generated by Trellis.
2. Read [Directory Structure](./directory-structure.md) before adding/moving platform files.
3. Read [Component Guidelines](./component-guidelines.md) before editing agent, skill, or command Markdown/TOML.
4. Read [Hook Guidelines](./hook-guidelines.md) before changing context injection or host hook configuration.
5. Read [State Management](./state-management.md) before changing task/session/context propagation.
6. Read [Type Safety](./type-safety.md) before editing JSON/TOML/TypeScript/Python contract boundaries.
7. If the change also touches `.trellis/scripts/`, read the backend runtime spec.

---

## Quality Check

For platform-file changes, use a combination of static checks and focused reads:

```bash
python -m compileall .claude/hooks .codex/hooks .trae/hooks .trellis/scripts
python ./.trellis/scripts/get_context.py --mode phase --step 2.1 --platform claude
python ./.trellis/scripts/get_context.py --mode phase --step 2.1 --platform codex
python ./.trellis/scripts/get_context.py --mode phase --step 2.2 --platform claude
python ./.trellis/scripts/task.py validate 00-bootstrap-guidelines
```

Also inspect the edited JSON/TOML/Markdown for syntax and frontmatter consistency. If editing `.pi/extensions/trellis/index.ts` or `.opencode/lib/trellis-context.js`, run the project's TypeScript checks only if a package manager setup exists; otherwise document that no JS/TS test harness is present in this workspace.
